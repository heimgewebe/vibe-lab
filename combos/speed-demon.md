# ⚡ Speed Demon

**Stil**: Guided YOLO | **Techniken**: Prompt Priming + Prompt Chaining | **Tool**: Cursor

## Beschreibung

Die Kombination für **maximale Geschwindigkeit mit Mindestqualität**: Guided YOLO gibt leichte Leitplanken, Priming setzt den Qualitätsstandard, und Chaining teilt die Aufgabe in verdauliche Häppchen.

## Rezept

### Schritt 0: Prime setzen
```
"Du bist ein erfahrener Full-Stack-Entwickler. Du schreibst 
sauberen, typisierten Code mit TypeScript. Du bevorzugst 
pragmatische Lösungen über Overengineering."
```

### Schritt 1: Scaffold (Guided YOLO)
```
"Erstelle eine [App-Typ] mit [Tech-Stack].
Leitplanken: TypeScript strict, feature-basierte Ordnerstruktur, 
Tailwind für Styling."
```

### Schritt 2: Features (Prompt Chaining)
```
Chain 1: "Erstelle das Hauptfeature: [Feature 1]"
Chain 2: "Füge hinzu: [Feature 2]"
Chain 3: "Verbinde alles und erstelle die Navigation."
```

### Schritt 3: Polish
```
"Mach die App hübsch – responsive Layout, Loading States, 
Error Messages. Keine neuen Features."
```

## Erfahrungsbericht

- **Zeitaufwand**: 30-60 Minuten für eine funktionierende App
- **Qualität**: Gut genug für Demos und MVPs
- **Geeignet für**: Hackathons, Proof-of-Concepts, Lernprojekte

## Wann NICHT verwenden

- Produktionscode
- Sicherheitskritische Anwendungen
- Langfristige Projekte
