# 📚 The Learning Loop

**Stil**: Pair-Programming mit KI | **Techniken**: Rubber-Duck-Prompting + Chain-of-Thought | **Tool**: Cursor/Copilot

## Beschreibung

Die Kombination für **maximales Lernen**: Die KI ist nicht nur Codegenerator, sondern Lehrer, Mentor und Diskussionspartner. Jeder generierte Code wird verstanden, nicht nur akzeptiert.

## Rezept

### Schritt 1: Problem vorstellen (Rubber Duck)
```
"Ich möchte [Technologie/Konzept] lernen. 
Erkläre mir zuerst die Grundkonzepte.
Dann lass uns gemeinsam ein kleines Projekt bauen."
```

### Schritt 2: Gemeinsam implementieren (Pair Programming)
```
"Lass uns Schritt für Schritt vorgehen.
Erkläre mir bei jedem Schritt:
- WAS wir machen
- WARUM wir es so machen
- WELCHE Alternativen es gäbe"
```

### Schritt 3: Vertiefen (Chain-of-Thought)
```
"Jetzt möchte ich verstehen, wie [Aspekt] im Detail funktioniert.
Denke Schritt für Schritt durch, was hier passiert."
```

### Schritt 4: Selbst ausprobieren
```
"Jetzt versuche ich es selbst. Gib mir nur Hints wenn ich festhänge."
```

## Erfahrungsbericht

- **Zeitaufwand**: 2-4x langsamer als reines Generieren
- **Lerneffekt**: Deutlich höher als Tutorials oder Dokumentation
- **Geeignet für**: Neue Sprachen, Frameworks, Paradigmen lernen

## Wann NICHT verwenden

- Wenn Geschwindigkeit wichtiger als Verständnis ist
- Für bekannte Technologien (dann ist es Zeitverschwendung)
