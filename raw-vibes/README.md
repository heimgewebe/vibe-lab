# raw-vibes/

Hier landen rohe Ideen, spontane Beobachtungen, Prompt-Fragmente und unfertige Hypothesen — **ohne Formzwang**.

## Was gehört hierhin?

- Flüchtige Session-Notizen
- „Ich glaube, X funktioniert besser als Y"
- Prompt-Entwürfe, die noch nicht getestet sind
- Qualitative Flow-Beobachtungen
- Alles, was du festhalten willst, bevor du es vergisst

## Was gehört hier NICHT hin?

- Strukturierte Experimente → `experiments/`
- Validierte Practices → `catalog/`
- Fertige Prompts → `prompts/adopted/`

## Regeln

Es gibt fast keine:

1. **Eine Datei pro Idee** (Markdown bevorzugt, aber nicht erzwungen)
2. **Kein Schema, kein Frontmatter, keine Pflichtstruktur**
3. **Kein CI-Check** — dieser Ordner wird nicht validiert
4. **Dateiname = kurzer Titel** (z. B. `chain-of-thought-beim-refactoring.md`)

## Und dann?

Wenn eine Idee reif genug ist, wird daraus ein Experiment:

```
raw-vibes/meine-idee.md  →  experiments/2026-xx-xx_meine-idee/
```

Das ist kein automatischer Prozess — du entscheidest, wann eine rohe Idee strukturiert genug ist, um getestet zu werden.

## Meta-Test: Macht die Struktur den Vibe kaputt?

> Dieser Ordner ist selbst ein Experiment: Senkt ein expliziter Rohraum die Einstiegshürde?
> Wenn raw-vibes leer bleibt, während Issues voll sind, stimmt etwas mit der Reibung nicht.
> Beobachtungen dazu gern hier oder als Experiment festhalten.
